# GitHub学习笔记
## 一、概述
**GitHub** 是全球最大的代码托管平台，基于 **Git** 版本控制系统构建。

作为开发者协作、开源项目以及企业研发的核心基础设施，GitHub 提供了以下关键能力：

- **代码托管与管理**：利用 Git 的分布式版本控制，安全存储和管理代码仓库。
- **协作与社区**：通过 Pull Request、Issues、Projects 等功能，支持高效的团队协作和开源贡献。
- **自动化与集成**：内置 GitHub Actions 等 CI/CD 工具，实现从代码提交到部署的全流程自动化。
- **企业级支持**：为企业提供安全合规、权限管理、高级审计等企业级功能，支撑大规模研发交付。
- **感想**：经过相关的背景了解到github好像是把全球热爱开发的人聚集在这一个平台，打破了信息的壁垒，在这个平台有很多小白，当然也有很多的“大神”，所以我们可以利用这些资源从而取长补短，更好的利用这层信息。
## 二、相关内容
### GitHub 核心功能

- **Issues**：任务跟踪与 Bug 反馈  
  用于记录待办事项、报告缺陷、讨论需求，并支持标签、里程碑和指派。

- **Pull Requests (PR)**：代码审查与合并  
  提供代码变更的对比、评论、逐行审查，通过后合并到目标分支，确保代码质量。

- **Projects**：项目看板管理  
  基于看板视图（Kanban）可视化工作流，支持自定义列、自动化状态同步，与 Issues 和 PR 联动。

- **Actions**：CI/CD 自动化  
  通过 YAML 定义工作流，实现自动构建、测试、部署等持续集成与持续交付流程。

- **Wiki / Discussions**：文档与社区交流  
  Wiki 用于项目文档的协同维护；Discussions 作为问答、公告和开放式讨论的社区空间。
