# HTML 基本语法笔记及注意事项

## 一、HTML 文档基本结构

```html
<!DOCTYPE html>                <!-- 声明文档类型为 HTML5 -->
<html lang="zh-CN">            <!-- 根元素，建议声明语言 -->
<head>
    <!--这里放置文档的元信息-->
    <meta charset="UTF-8">     <!-- 字符编码声明，避免乱码 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- 移动端适配 -->
    <title>页面标题</title>    <!-- 显示在浏览器标签栏 -->
</head>
<body>
    <!--这里存放置页面内容-->
    <h1>这是一个标题</h>
    <p>这是一个段落。</p>
    <ul>无序列表</ul>
    <ol>有序列表</ol>
    <a href="https://www.example.com">这是一个链接</a>
    <!-- 页面可见内容 -->
</body>
</html>
```

---

## 二、标签语法规则

| 类型 | 写法 | 示例 |
|------|------|------|
| **双标签** | `<标签名>内容</标签名>` | `<p>段落</p>` |
| **单标签（自闭合）** | `<标签名>` 或 `<标签名 />` | `<br>`、`<img src="...">` |
| **嵌套** | 正确闭合，不可交叉 | `<div><p>正确</p></div>` ❌ `<div><p>错误</div></p>` |

> ✅ 推荐单标签不写 `/`（如 `<br>`），但 `<img>`、`<input>` 等加 `/` 也有效，不影响解析。

---

## 三、属性书写规范

```html
<标签名 属性名="属性值" 属性名='属性值' 属性名=属性值>
```

- **属性值建议加引号**（双引号为主，单引号也可），避免特殊字符或空格导致解析错误。
- **布尔属性**（如 `disabled`、`checked`）可只写属性名，表示 `true`：
  ```html
  <input type="checkbox" checked>   <!-- 表示选中状态 -->
  ```

---

## 四、常用元素分类（简记）

| 类别 | 特点 | 示例 |
|------|------|------|
| **块级元素** | 独占一行，宽高可设 | `<div>`、`<h1>`、`<p>`、`<ul>`、`<section>` |
| **内联元素** | 同行显示，宽高由内容撑开 | `<span>`、`<a>`、`<strong>`、`<img>`（特殊内联替换元素） |
| **内联块级** | 同行显示，可设宽高 | `<img>`、`<input>`、`<button>` |

> ⚠️ 块级元素内可嵌套内联元素，但内联元素内不应嵌套块级元素（`<a>` 标签在 HTML5 中可包裹块级元素，但需注意交互层级）。

## 五、注释与特殊字符

```html
<!-- 这是注释，不会在页面中显示 -->
```

**常用转义字符**（避免与 HTML 语法冲突）：

| 字符 | 实体名称 | 说明 |
|------|----------|------|
| `<` | `&lt;` | 小于号 |
| `>` | `&gt;` | 大于号 |
| `&` | `&amp;` | 与符号 |
| `"` | `&quot;` | 双引号 |
| `'` | `&apos;` | 单引号（部分浏览器支持） |
| 空格 | `&nbsp;` | 不换行空格 |

## 六、核心注意事项

### 1. 标签必须正确闭合
- 双标签忘记闭合会导致布局错乱，如 `<div>` 未闭合会影响后续元素。
- 自闭合标签虽不强制 `/`，但建议统一风格。

### 2. 嵌套顺序严格遵循
错误示例：
```html
<b>加粗<i>倾斜</b>错误</i>   <!-- 交叉嵌套 → 解析异常 -->
```
正确应：
```html
<b>加粗<i>倾斜</i></b>
```

### 3. `<title>` 必不可少
影响 SEO、浏览器标签显示、分享卡片等。

### 4. 字符编码统一
`<meta charset="UTF-8">` 应放在 `<head>` 最前面，避免页面出现乱码。

### 5. 图片必须添加 `alt` 属性
增强可访问性，图片加载失败时显示替代文本：
```html
<img src="photo.jpg" alt="描述图片内容">
```

### 6. 链接的 `href` 与交互
- `href="#"` 会跳转到页面顶部，可用 `javascript:void(0);` 或省略 `href` 配合 `button`。
- 外部链接建议加 `target="_blank" rel="noopener noreferrer"` 保证安全。

### 7. 块级与内联元素混淆
- 不要在内联元素（如 `<span>`）内直接嵌套块级元素（如 `<div>`），会导致布局异常。
- 可通过 CSS 的 `display` 属性改变显示行为，但 HTML 结构应保持语义正确。

### 8. 语义化标签优先
使用 `<header>`、`<nav>`、`<main>`、`<article>`、`<footer>` 等替代泛滥的 `<div>`，利于 SEO 和屏幕阅读器。

### 9. `id` 必须唯一，`class` 可重复
`id` 用于 JavaScript 或锚点定位，同一页面不可重复；`class` 用于样式复用。

### 10. 大小写统一小写
虽然 HTML 标签不区分大小写，但 W3C 规范推荐全小写，且 XHTML 要求小写。

## 七、常见错误速查表

| 错误现象 | 可能原因 |
|----------|----------|
| 页面出现乱码 | 缺少 `<meta charset="UTF-8">` 或文件编码不一致 |
| 样式/脚本不生效 | 标签未闭合、属性值未加引号、路径错误 |
| 布局错位 | 标签交叉嵌套、块级与内联嵌套不当 |
| 移动端缩放异常 | 未添加 `viewport` meta 标签 |
| 链接跳转异常 | `href` 为空或 `#` 导致页面刷新 |
