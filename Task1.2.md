## 标题与层次结构

使用 `#` 表示标题，从一级到六级。保持层级清晰，便于 GitHub 自动生成目录。

```markdown
# 一级标题（通常用于 README 顶部）
## 二级标题（主要章节）
### 三级标题（子章节）
**加粗**：用于突出重要概念  
*斜体*：用于术语或轻微强调  
***加粗斜体***：用于特别强调  
~~删除线~~：表示已废弃或错误内容  
`行内代码`：用于命令、文件名、参数
- 无序列表项 1
- 无序列表项 2
  - 嵌套项

1. 有序列表第一项
2. 有序列表第二项

- [x] 已完成任务
- [ ] 未完成任务
| 功能 | 描述 | 状态 |
|------|------|------|
| 用户认证 | 支持 OAuth 2.0 | ✅ 已完成 |
| 数据导出 | 支持 CSV 格式 | 🚧 开发中 |
<details>
<summary><b>📋 点击展开更新日志</b></summary>

- **v2.0.0** - 2024年重构，增加新特性
- **v1.0.0** - 首次发布

</details>
