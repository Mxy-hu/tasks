# HTML

## 骨架与区域划分
- 整体使用**块状元素**划分区域，每个块状元素代表一个独立的内容区块。
- 区块内部填充具体内容。

## 关键结构点

### 导航栏
- 使用**语义标签**（如 `<nav>`）包裹导航栏。
- 构建**水平菜单**的列表（ `<ul>` + `<li>`）。
- 使用 `class="ctn"` 让 **Contact Us** 按钮高亮显示。

### 左右栏布局
- 由 CSS 实现左右两栏布局。
- 包含重要的标题。
- 两个 `<标签>` 作为按钮（使用 `<button>` ）。

### 服务器展示网格
- 容器：`.grid-wrapper`
- 内部多个 `.grid-box`，用于展示四个服务器。
- 每个服务器项目：
  - 使用 `.prj-img` 包裹图片。
  - 包含一个悬停时显示的 `.details` 遮罩层。
  - 使用图标增强视觉效果。

# CSS 

## 1. 样式计算（Recalc Style）

- **全局重置** `* { margin:0; padding:0; box-sizing:border-box; }` 影响所有元素。
- **字体 & 颜色**：`body` 设置 `Poppins` 字体，`h1`、`p`、`a` 分别定义颜色。

## 2. 布局（Layout）

浏览器根据元素的显示类型（`display`、`position`、`flex`、`grid` 等）计算几何位置和尺寸。

- **导航栏**：`.navbar` 为 `absolute` 定位，脱离文档流，位于页面左上角；内部 `<ul>` 使用 `flex` 和 `text-align:right` 将菜单项右对齐。
- **头部区域**：`.row` 为 `flex` 容器，`.header-content` 和 `.hero-img` 各占 `50%` 宽度，内容水平排列。
- **服务部分**：`.grid-wrapper` 使用 `display: grid` + `repeat(auto-fit, minmax(250px, auto))`，会根据容器宽度自动换行，生成响应式网格。
- **数字部分**：`.number .grid-box` 使用相同的网格模式，并设置了 `hover` 时的 `translate(-20px)` 变换。
- **项目展示**：`.projects-images` 网格每列最小 `350px`，图片容器 `.prj-img` 设置 `overflow: hidden`，悬停时覆盖层 `.details` 通过 `transform: scaleX(0)` → `scaleX(1)` 实现从左到右展开动画。

## 3. 绘制（Paint）

浏览器将每个元素的视觉部分（颜色、背景、边框、阴影等）填充到对应的图层中。

- **渐变背景**：多处使用 `var(--gradient-*)` 定义线性渐变，如 `.mobileapp`、`.webdesign` 等，绘制时会生成渐变图像。
- **阴影**：`.grid-box` 和 `.mockup-img img` 使用了 `box-shadow`，增加立体感。
- **图片**：`.hero-img img`、`.circle img` 等通过 `animation: scaleup` 实现无限缩放动画，动画属性在绘制阶段会被持续更新。

## 4. 合成（Composite）

- `transform`：`.number .grid-box:hover` 的 `translate()`、`.team-men:hover` 的 `translateY()`、`.prj-img:hover .details` 的 `scaleX()` 等，这些变换在合成器线程处理。
- `animation`：`.grid-box img` 和 `.circle img` 的 `scaleup` 动画同样使用 `transform: scale()`，避免重排。
- 未使用 `top`/`left` 等触发重排的属性。

# JavaScript 

 **仅依赖 CSS 效果**  
   页面中的视觉变化完全由 CSS 提供：
   - 悬停效果（`.ctn:hover`、`.Learnmore:hover`、`.prj-img:hover .details` 等）
   - 过渡动画（`.number .grid-box:hover` 的 `transform`、`.team-men:hover` 的 `translateY`）
   - 无限循环动画（`.grid-box img` 和 `.circle img` 的 `scaleup` 关键帧动画）


## 页面中的动态元素（均为 CSS 实现）

| 元素 | 效果 | 触发方式 |
|------|------|----------|
| 导航栏 `ctn` 按钮 | 背景渐变变化 | `:hover` |
| 服务网格卡片 | 阴影、缩放动画 | `:hover` + `transform` |
| 项目图片覆盖层 | 从左向右展开 | `:hover` + `transform: scaleX()` |
| 团队成员卡片 | 向上平移 | `:hover` + `transform: translateY()` |
| 团队成员社交图标 | 放大并变色 | `:hover` + `transform: scale()` |
| 圆形背景图 & 服务图标 | 周期性缩放 | 无限 `animation` |
所以项目中的JS并没有，主要是依靠CSS来实现的动态变化

# 困难
* **困难一**：代码错误，但是很难找到原因，我认为这是最大的困难因为代打是非常冗长的，所以一行一行的检查基本上是不怎么可能的事情，查找之下其中安插IDE插件是一个很好的方法，可以自动检测到其中的语法错误。Vscode就是这样安装插件，然后打开对应的CSS文档，错误会有红色波浪线，鼠标悬停时会有错误的详情，从而找到详细的解决方法。
* **困难二**：打开代码所对应的网址，图片的有一部分是缺陷的，通过查找这是响应式布局的缺陷，部分尺寸使用固定 px，在小屏幕上可能导致溢出，.hero-img img 使用 margin-left: 80px，在窄屏幕下会超出父容器，网格 grid-template-columns: repeat(auto-fit, minmax(250px, auto)) 在宽屏下合理，但是针对手机可能就不行了。所以使用使用相对单位（vw、%、rem）替代固定 px，避免溢出是一个比较好的方法。
