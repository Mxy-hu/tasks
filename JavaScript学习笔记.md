# JavaScript学习笔记
---
## 学习背景
在我的了解下，JavaScript的应用其实是非常广泛的，甚至可以作为第一语法来学习，不需要所谓的其他编程语言来支撑，学习JavaScript需要一定的编程逻辑思维，极少的微积分，线性代数，响应式交互需要该语言作为支撑，仅仅需要一些CSS和HTML知识，是非常便利的。
## 学习内容目录
* **基础语法**
* **数组与对象**
* **DOM操作**
* **异步编程**
## 学习内容
* **1基础语法**：其中包含输出与注释、变量、数据类型、运算符、条件判断、循环、函数。
**感想**：语法基础是一定不可缺的，但是需要多敲代码才能形成所谓的记忆以及习惯，例如=和===容易混用是其中我认为一个细小的点，还有其中的for循环需要明白从哪里开始，走到哪一步了，还有每一步是怎么变化的，我认为如果自己能做一些更多的针对练习结果会更好
* **数组与对象**：数组-有序列表、对象-键值对集合、数组的常用高阶方法
**感想**：数组中的有序、索引连续、自动维护length、有丰富的迭代方法，适合列表数据。普通对象其中的无序、键值对，适合记录数据，例如关键转换为Array.from,关键检测为Array.isArray.
* **DOM操作**：获取元素、修改内容和样式、添加/删除元素、事件监听
**感想**：是游览器将HTML文档解析成一个树状结构的一个工具，每个节点都是一个对象，JavaScript通过游览器提供的对象以及所对应的方法来读取、修改、删除、创建DOM节点，并响应用户事件，从而实现动态可交互的网页，这两者的结合是前端工程师“灵魂”的呈现，可以决定一个元素什么时候开始什么时候结束以此来响应用户的动作，以此使得静态网页更加的流畅。
* **异步编程**：计数器、Promise Fetch\JSON转换
**感想**：我们平时不会做前面的题的时候我们会跳过这个题，先去做后面的题再来做前面的题，不会让这样一个题影响我们的进程。在我看来异步编程也是如此，因为一行一行的运行太慢了，并且如果前面出了错后面也不运行了这肯定是不行的，异步编程可以让错处放在一边先解决后面的问题再来解决前面的问题。在我的之后了解下许多的前端难题（例如竞态条件、加载状态管理、性能优化等等）都需要异步编程来解决，异步编程在我心目中的位置还是挺重的。
